十四、容易踩的坑

第一个坑：以为 UBO 绑定一次就所有 shader 都自动知道。

不是的。

UBO 绑定到 binding point：

shader 的 Uniform Block 也要绑定到同一个 binding point：

这两端都连上，数据才通。

第二个坑：忘记每个 shader 都要绑定 Uniform Block。

如果四个 shader 都有 Matrices，那么每个 shader 的 Matrices block 都要绑定到同一个 binding point。

第三个坑：std140 padding 算错。

尤其是：

这些类型在 std140 中经常有额外对齐和填充。
如果 C++ 侧结构体没有对齐好，shader 读到的数据就会错位。

第四个坑：把变化频率很高、每个物体不同的数据也塞进共享 UBO。

例如 model 矩阵每个物体都不同。
如果强行放进共享 UBO，反而会让更新逻辑变复杂。

第五个坑：手动写 gl_FragDepth 却忘了性能影响。

写 gl_FragDepth 可能影响提前深度测试。
如果只是普通渲染，不要轻易手动改深度。

第六个坑：开启面剔除后还期望 gl_FrontFacing 处理被剔除的背面。

被面剔除丢掉的三角形不会产生片段。
所以片段着色器也就没有机会判断它们。
