这篇文章记录一下我重新搭建个人博客的过程。整个方案并不复杂，核心就是两件事：使用 Hexo 负责本地内容管理与静态页面生成，使用 GitHub Pages 负责博客的托管与发布。

对个人博客来说，这套方案有几个很直接的优点：

  • 本地写作体验比较轻，适合长期维护
  • 生成的是静态文件，部署成本低
  • 可以直接挂在 username.github.io 这样的域名上
  • 配合 Git 和 GitHub，备份与迁移都比较方便

为什么选择 Hexo

Hexo 是一个经典的静态博客框架。对我来说，它最大的价值不是“功能特别多”，而是结构简单、可控，而且改主题、改配置、改部署方式都比较直接。

我需要的是一个可以长期记录技术内容的博客，而不是一个过于复杂的后台系统。Hexo 刚好适合这种场景。

主题与风格

博客当前使用的是 hexo-theme-desktop 这个主题，并且在此基础上继续做了一些调整：

  • 清理掉旧主题遗留配置
  • 调整为更适合中文阅读的字体策略
  • 修改了首页卡片文案和整体色彩气质
  • 修复了空站点状态下首页无法正常展示的问题

对个人博客来说，主题不是一次性决定的东西。先跑通，再慢慢打磨，比一开始就追求完全定稿更重要。
