学到 LearnOpenGL 的光照部分时，我第一次明显感觉到“图形学开始像图形学了”。

前面的窗口、三角形、着色器、纹理、坐标系统和摄像机，更多是在回答这些问题：

  • 程序怎么跑起来
  • 顶点怎么送进 GPU
  • 物体怎么出现在三维空间里
  • 相机为什么能移动

而光照这一章开始回答的是另一类问题：

  • 为什么同样是一个立方体，有的看起来像塑料，有的像金属
  • 为什么物体会有明暗变化，而不是一整块纯色
  • 为什么高光会随着视角移动
  • 为什么不同位置、不同类型的光源，会让同一个场景呈现完全不同的气氛

这一篇笔记把整个光照主线串在一起记录，对应的本地项目是：

  • projects/learnopengl/learnopengl/09_lighting_colors
  • projects/learnopengl/learnopengl/10_basic_lighting
  • projects/learnopengl/learnopengl/11_materials
  • projects/learnopengl/learnopengl/12_lighting_maps
  • projects/learnopengl/learnopengl/13_light_casters
  • projects/learnopengl/learnopengl/14_multiple_lights

如果只把它们看成六个小节，学习时很容易碎掉；但如果把它们看成一条连续演化的链路，逻辑会非常清楚：

  1. 先建立“物体颜色会受光源颜色影响”的直觉
  2. 再建立一套最基本的光照模型
  3. 然后把“物体是什么材质”引入进来
  4. 再把材质参数从常量升级成贴图
  5. 再讨论光到底是从哪里来的、怎么衰减、怎么形成聚光
  6. 最后把不同类型的光源同时放进一个场景里

也就是说，这一章真正完成的是一次认知升级：

从“给物体指定颜色”，到“由光照模型决定物体最终呈现出来的颜色”。

为什么光照应该整章连起来学

这一章的每一节都不是孤立的。

“颜色”看起来最简单，但它在帮我们建立一个最基本的起点：物体颜色和光源颜色之间是会相互作用的。

“基础光照”则是在这个起点上，第一次把明暗变化拆成了环境光、漫反射和镜面高光。

“材质”进一步说明：不同物体面对同一束光，呈现出来的效果并不相同，因为物体本身的反射属性不同。

“光照贴图”又说明：这些材质属性不仅可以是一个常量，还可以是纹理，也就是说，物体的不同区域可以有不同的反射特征。

“投光物”开始讨论光源类型。原来光不一定只是“一个位置 + 一个颜色”，它还可能是平行光、点光源、聚光灯，而且会随距离衰减。

最后“多光源”把这一切收束起来：现实场景里通常不会只有一盏灯，真正的渲染程序要把多种光源的贡献叠加在一起。

所以这整章最好不要拆碎记，而应该作为一条完整的主线来理解：

“物体如何根据自身材质、表面朝向、观察方向以及场景中的多个光源，计算出最终颜色。”

第一部分：颜色并不是贴上去的，而是算出来的

光照章节的第一个起点其实很朴素。

在“颜色”这一节里，物体片元着色器通常只做了一件事：

最终颜色 = lightColor * objectColor

这还不是完整的光照模型，但它已经足够帮助我建立一个重要直觉：

  • 光源本身有颜色
  • 物体本身也有颜色
  • 屏幕上看到的结果，往往不是“直接等于物体颜色”，而是两者共同作用的结果

这一节通常还会在场景里放一个小白色立方体，专门表示光源位置。这个做法在后面看起来非常自然，因为一旦开始做真正的光照计算，我们就不再只是“知道有一盏灯”，而是会明确地知道“灯在场景中的什么位置”。

换句话说，“颜色”这一节虽然简单，但它完成了两件很重要的铺垫：

  • 把“物体”和“光源”分开
  • 把“颜色是被光照影响的结果”这个概念先立住

第二部分：基础光照真正建立了明暗关系

到了“基础光照”，事情才真正进入图形学核心。

这一节最重要的不是代码量变大了，而是第一次开始认真回答：

“一个片段为什么该亮，为什么该暗？”

LearnOpenGL 这里使用的是经典的 Phong/Blinn-Phong 入门思路里的那条主线：把光照拆成三部分。

1. 环境光 Ambient

环境光可以理解成场景里总有一点基础亮度，不至于让背光面彻底黑掉。

它并不追求物理严格，更像是入门阶段为了让场景更自然的一种基础补光。

2. 漫反射 Diffuse

漫反射最关键的量是：

dot(normal, lightDir)

它表达的是“表面朝向”和“来光方向”之间的对齐程度。

  • 如果法线和光线方向几乎一致，说明光打得比较正，表面更亮
  • 如果夹角很大，说明光打得比较斜，表面更暗

这一步让立方体第一次真正具有了“面朝光的一侧更亮，背光的一侧更暗”的立体感。

3. 镜面高光 Specular

镜面高光开始把观察者也引入计算。

它不再只关心光从哪里来，还关心：

  • 我现在从哪个方向看这个表面
  • 反射方向是不是接近我的视线方向

这也是为什么高光会随着视角移动。它不是贴在物体表面上的一块白斑，而是由“光线方向、表面法线、观察方向”共同决定的结果。

4. 法线为什么这么重要

这一节里最关键的新输入其实不是光源颜色，而是法线。

法线描述的是表面朝向。没有法线，就没有办法判断：

  • 这一块表面是不是朝向光源
  • 光在这里应该有多强的漫反射
  • 高光应该出现在哪里

这也是为什么这一章开始，顶点数据不再只是位置，还要额外包含法线。

5. 为什么所有向量必须放在同一个空间里

这一节还有一个非常容易忽略但极其关键的原则：

参与光照计算的量，最好全部放在同一个坐标空间里。

例如：

  • FragPos
  • Normal
  • lightPos
  • viewPos

如果其中一部分在局部空间，另一部分在世界空间，那么点乘、方向差值这些结果就会失去意义。

在我们的实现里，基础光照之后的代码基本都选择在世界空间中完成这件事。这一点会贯穿后面所有小节。

第三部分：材质回答了“这是什么东西”

“基础光照”让我们知道了物体为什么会亮和暗，但它默认了一个很粗糙的前提：所有物体的反射行为差不多。

现实里当然不是这样。

  • 木头的高光弱
  • 金属的反光强
  • 塑料会有明显高光，但又不像金属那么锐利

这时候，“材质”这一节就出现了。

它做的事可以概括成一句话：

“把光照公式里原本写死的颜色和高光强度，拆成属于物体自己的材质参数。”

通常我们会把材质拆成：

  • ambient
  • diffuse
  • specular
  • shininess

前面三个控制不同光照分量如何响应，shininess 则控制高光有多集中。

这里最容易建立的直觉是：

  • specular 越强，高光越明显
  • shininess 越大，高光范围越小，但更尖锐

这一节很像在说：

基础光照告诉你“光照怎么算”，而材质告诉你“这个物体愿意以什么方式响应这些光”。

如果说前一节讨论的是光照模型的结构，那么这一节讨论的就是物体自身的反射个性。

第四部分：光照贴图把“材质常量”升级成了“材质分布”

学到“材质”之后，一个很自然的问题就会出现：

难道整个物体的漫反射、镜面反射都只能是一个固定值吗？

如果是一个木箱，木头纹理不同区域看起来并不完全一样；有些金属边角更容易出现高光，有些区域则更暗、更粗糙。

于是“光照贴图”把材质再往前推了一步：

  • 漫反射不再只是常量，而可以来自 diffuse map
  • 镜面反射不再只是常量，而可以来自 specular map

这一步的重要意义在于：

材质不再只是“整个物体的一份配置”，而变成了“物体表面每个位置都可能不同的属性分布”。

这时候，纹理坐标重新变得非常重要，因为片元着色器需要根据 TexCoord 去查询：

  • 这一点表面的基础颜色是什么
  • 这一点表面的镜面反射强度是多少

如果说“材质”是在回答“这是什么材料”，那么“光照贴图”就在回答：

“这个材料在物体表面的不同区域，具体是怎样分布的。”

这也是光照章节第一次真正把“纹理”和“光照”深度结合起来。

第五部分：投光物开始讨论“光从哪里来”

前面几节更多是在讨论：

  • 物体怎样接收光
  • 物体怎样根据材质响应光

但“投光物”开始真正讨论光源本身。

这时候我们不再满足于说“场景里有一束光”，而是要区分：

  • 这盏光是沿某个固定方向照下来的吗
  • 它是从某个具体位置向四周发散的吗
  • 它会不会像手电筒一样只照亮一小块区域
  • 它会不会随距离变远而变暗

这一节最重要的变化，是把“光源类型”纳入了程序结构。

1. 方向光 Directional Light

方向光可以理解成来自非常远处的光，例如阳光。

它的特点是：

  • 只有方向
  • 不强调具体位置
  • 所有片段接收到的来光方向大致相同

这类光很适合做场景的基础主光。

2. 点光源 Point Light

点光源则强调一个明确的位置。

它从某个点向四周发散，因此离光源近的地方更亮，离得远的地方更暗。

这时候就引入了衰减：

1.0 / (constant + linear * d + quadratic * d^2)

这条式子非常重要，因为它让点光源不再“照亮全世界”，而更像现实中会随着距离减弱的灯泡。

3. 聚光灯 Spot Light

聚光灯除了有位置，还有方向，并且有一个照射角范围。

这类光最容易让人联想到手电筒。

在我们的实现里，它还和摄像机绑定在一起，于是你移动和转头时，聚光灯的亮区也会一起移动。

如果再加入内外切角和插值，就能做出柔和边缘，而不是生硬地“圈内亮、圈外黑”。

这一节真正带来的理解是：

光照不仅仅是“算一个公式”，还要考虑光源的空间属性和照射方式。

第六部分：多光源把整章真正收束了

到了“多光源”，这一章的逻辑终于闭环了。

现实中的场景很少只存在一盏灯。一个房间里可能同时有：

  • 从窗外照进来的方向光
  • 天花板上的多个点光源
  • 玩家手上的手电筒聚光灯

多光源这一节做的事情，其实就是把前面已经学过的各种光源组织起来，然后把它们对同一个片段的贡献叠加。

这一节最重要的不是新的数学，而是新的组织方式。

通常会做这些事：

  • 为方向光单独定义结构体
  • 为点光源定义结构体数组
  • 为聚光灯定义结构体
  • 分别写出 CalcDirLight、CalcPointLight、CalcSpotLight
  • 在片元着色器里把它们的结果相加

到了这一步，片元着色器的核心就从“单灯光照公式”变成了“多种光照贡献的组合器”。

这也是为什么这一节看起来更像工程问题，而不只是数学问题：

你不只是要会算，还要会组织 shader 代码，让它能清晰地支持多种光源并存。

把整个光照章节压缩成一条主线

如果让我把这一整章压缩成最短的一条主线，我会这样概括：

  1. 先认识到物体颜色会受光源颜色影响
  2. 再用环境光、漫反射和镜面高光建立最基本的明暗模型
  3. 然后用材质参数控制不同物体如何响应光
  4. 再用贴图让材质从“常量”变成“分布”
  5. 再用方向、位置、衰减、聚光角度去描述不同类型的光源
  6. 最后把多种光源的结果叠加，形成一个完整场景

换句话说，这整章是在不断回答同一个问题，只是回答得越来越完整：

“一个片段的最终颜色，到底是怎样被算出来的？”

这一章我最想真正记住的几个点

学完整章后，我觉得最值得反复回想的不是某个单独 API，而是下面这些原则。

1. 光照计算本质上是“方向关系”的计算

无论是漫反射还是镜面高光，核心都离不开方向之间的关系：

  • 法线和光线方向的关系
  • 观察方向和反射方向的关系

也就是说，光照不是在“给颜色”，而是在“根据几何关系计算颜色”。

2. 法线是表面朝向，不是装饰数据

法线一旦不对，明暗关系就会整体失真。

尤其是在后面有缩放变换时，法线不能简单地直接乘 model，而需要使用法线矩阵。这一点在基础光照之后就应该牢牢记住。

3. 材质和贴图并不是两套体系，而是一套体系的不同层次

材质先把物体的反射行为抽象成参数。

贴图则让这些参数在表面空间中变得更细致。

所以贴图不是额外附加的一层装饰，而是材质表达方式的升级。

4. 多光源不是推翻前面的公式，而是把它们加起来

这一点很重要，因为它说明：

后面的复杂，并不是因为前面的内容失效了，而是因为前面的每一部分都继续有效，只是需要同时存在。

5. 光照章节的本质，是把“几何世界”和“视觉效果”第一次真正连接起来

在坐标系统和摄像机阶段，我们更多是在组织空间。

而到了光照阶段，这个空间终于开始产生视觉上的“质感”。

这也是为什么光照是一个非常关键的分水岭：从这里开始，画面不再只是“有东西在那儿”，而开始变得“像一个被看见的世界”。

这一路最自然的学习顺序为什么是对的

现在回头看 LearnOpenGL 这条光照学习顺序，我觉得它设计得非常合理。

它没有一上来就把所有公式和所有光源堆在你面前，而是一步一步扩展问题：

  • 先分清光源和物体
  • 再引入法线和明暗
  • 再引入材质差异
  • 再引入贴图
  • 再引入不同光源类型
  • 最后收束到多光源

这意味着我们不是在硬背一个大公式，而是在逐步构建一套完整的理解框架。

这也是为什么把整章串成一篇笔记会比单独看每一节更有效：

因为你看到的将不再是六个小知识点，而是一套不断扩展、不断变完整的渲染思路。

小结

如果说入门阶段解决的是“怎么把东西画出来”，那么光照阶段解决的就是：

“怎么让它看起来不再只是画出来，而是真正开始有体积、有材质、有空间中的视觉逻辑。”

从“颜色”到“多光源”，我最终学到的并不是几个孤立的 shader 技巧，而是一条非常完整的认识路径：

  • 颜色会受到光源影响
  • 表面朝向会决定明暗
  • 观察方向会决定高光
  • 材质会决定反射特性
  • 贴图会决定这些特性在表面上的分布
  • 光源类型会决定光如何投射到场景中
  • 多光源会共同塑造最终画面

到这里，OpenGL 的画面才真正开始从“几何正确”走向“视觉上更可信”。

而这也正是我觉得光照这一章最有价值的地方。
